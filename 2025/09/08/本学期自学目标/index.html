数据库（一，引入；关系模型和代数）
数据库管理系统（DBMS）： DBMS 是一种允许应用程序在数据库中存储和分析信息的软件。 DBMS 允许根据某些数据模型来定义，创建，查询，更新，管理数据库。 数据模型： 是一种描述数据库中数据的概念的集合： 实例：关系，NoSQL(键/值，文档，图形)，数组/矩阵/向量(用于机器学习) 关系数据模型： 关系数据模型定义了基于关系的数据库抽象概念，以避免更新数据库而要重写代码的开销。 关系模型三个原则： 1，用简单的数据结构来存储数据； 2，物理存储方式由 DBMS 的具体实现来决定。 3，只通过 sql 语言接受数据，DBMS 找出来最佳的执行策略。 关系模型定义三个概念： 1，结构：关系的定义及其内容。关系具有的属性，以及这些属性可以包含的值 2，完整性：确保数据库的内容满足约束。如：year 属性的值必须均为数字 3，操作：如何访问和修改数据库的内容 元组是关系中的一组属性值。每个属性都可以有特殊值 NULL，这表示该属性是未定义的。 具有 n 个属性的关系被称为 n 元关系。n 元关系相当于具有 n 列的表。 key: 主键： 一个关系的主键独一无二地标识...

堆和优先队列

发表于2025-09-09|更新于2025-09-09
|总字数:818|阅读时长:3分钟|浏览量:

优先级队列：

是一种ADT，定义了一下的接口：

  • 插入：向队列中加入一个带有优先级的元素；
  • remove操作：提取出队列中优先级最高的元素，并删除。

堆的引入：

操作\数据结构 有序数组 hash数组 bushy BST
add O(N) O(1) O(logN)
get_smallest O(1) O(N) O(logN)
remove_smallest O(N) O(N) O(logN)

从现在看来实现优先级队列的最优结构是BST。

但是否可以对BST进行进一步的优化？

我们知道BST中，左儿子 <= 父节点，我们若想要得到最小值，我们还要遍历一遍左子树[ O(logN) ]；那么我们可不可以在这个的基础上进行优化，定义一种min-heap，将优先级最高的作为父节点，那么我们进行提取时就能实现为O（1）的复杂度。

堆：

以min-heap为例，属性如下：

  • 每个节点都小于等于它的子节点；
  • 若在叶子节点处缺少项目，所有节点都进可能偏左。

如何在这种堆模式下，实现以上的三种操作呢？

  • add:

    将新值临时添加到堆的末尾，再按照规定的排序方式，递归地插入到应该所处的位置。

  • getSmallest：

    返回根节点即可。

  • removeSmallest：

    将堆的最后一项交换到根节点，再递归地调整树的结构即可。

第一章：计算机系统漫游

一，信息由字节表示：

无论是cpp文件亦或是其他文件，程序中的每一个文本字符都是由ASCII码表示的，例如hello.c文件由ASCII文本表示如下：

![屏幕截图 2025-09-02 181956](C:\Users\LENOVO\Pictures\Screenshots\屏幕截图 2025-09-02 181956.png)

像以上的文件被称为文本文件，所有其他文件均被称为二进制文件。

二，程序编译过程的不同形式：

系统上helllo.c文件的运行：

每条C语句被转化为低级机器语言指令，这些指令再按照称为可执行目标程序的格式打包，以二进制的磁盘文件形式存放。（目标程序文件也被称为可执行目标文件）

从源文件到目标文件的转化是由编译器驱动程序完成的。编译过程则可分为四个阶段，执行四个阶段的程序

（预处理器，编译器，汇编器，链接器）一起构成了编译系统。

![屏幕截图 2025-09-02 183128](C:\Users\LENOVO\Pictures\Screenshots\屏幕截图 2025-09-02 183128.png)

过程解释：

预处理阶段：

根据#子开头的命令，引用并插入库文件内容，得到对应的扩展文件；

编译阶段：

将文本文件hello.i翻译为文本文件hello.s，其中包括一汇编语言程序。

汇编语言的作用：将不同编译器编译的不同语言，转化为通用的汇编语言进行输出。

汇编阶段：

将hello.s翻译为机器语言指令，并将其打包为可重定位目标程序的格式—-hello.o，

链接阶段：

将printf函数进行解析，链接添加进hello.o文件中，此时才可最终执行。

第二章：信息的表示和处理

信息的存储：

程序将内存视作一个大型数组，数组的元素由一个个字节(Byte)组成，每个字节都由唯一的数字表示，称为地址。所有地址的集合就称为虚拟地址空间。

  • 字节：

    一个字节由8位组成。取值范围从00000000 ~ 11111111，即为0 ~ 255。这种一位位表示的方式称为位模式。

  • 十六进制的引入：

    1. 避免了二进制的冗长与十进制的麻烦：以 0 ~ 9与A ~ F构成。在C语言中，十六进制以0X开头（大小写随意）
    2. 便于与二进制进行转换：直接将每一位数字以二进制表示即可。注意当总位数不是4的倍数时，需要用0进行补齐。

  • 进制的转换：

    1. 2的n次方转换为二进制，即是1后面跟了n个0

    2. 二进制与16进制的转换：

      对于二进制 1000…0(n个0)，采取n = i + 4 * j 的方法。

      即表示该数由 j 个0与一个 i 组成。

    3. 十进制与十六进制的转换：

      采用辗转相除的方法；

  • 字长(Words):

    字长决定了在虚拟机中能够存储的最大位到多少。

    对于一个字长为w的机器，虚拟地址的范围是 0 ~ 2^w - 1。

    现常见的机器为32位与64位两种：

    其中32位的文件，两种机器均可运行（因为64位机器大都实现了向下兼容）；而64位的文件只能运行在64位机器上

    各类型数据的字节长：

  • 字节的存储顺序：

    1. 大端法：从高位往低位存储：

      小端法：从低位往高位存储：

  • ​ 布尔代数：

    • ~ （非）：

      ~0 = 1； ~1 = 0；

    • &（与）：

      0 & 0 = 0； 0 & 1 = 0； 1 &1 = 1；

    • |（或）：

      0 | 0 = 0；0 | 1 = 1；1 | 1 = 1；

    • ^（异或）：

      两参数相同时为0；不同时为1；

  • 掩码运算（Masking Operations):

    通过位运算可以得到特定的掩码序列。

    例如想要得到 0X89ABCDEF的最低有效字节：

  • ​ 位移运算：

    • 左移：左移n位即是在右端补上n个0

    • 右移：分为逻辑右移和算术右移：

      逻辑右移时，向高位补0；

      算术右移：

      ​ 当最高位是0时，向最高位补0；

      ​ 当最高位是1时，向最高位补1；

      当数值为无符号数是，右移一定是逻辑右移；

      当数值为有符号数时，右移一定是算术右移；

整数表示：

补码：

整数通常用补码进行表示：

正数：直接二进制表示；

负数：其补码=对应正数的二进制，取反后再加1；

无符号数的编码(Unsigned):

采用一个向量表示vector X = [a1, a2, a3….];

使用转换函数B2U后得到：

对于B2U([1011]) = 1*2^3 + 1 * 2^1 + 1 * 2^0;

无符号数 的值就是数值为1的单位的对应长度之和。

有符号数的编码：

采用补码的形式。

对于向量X = [a1, a2, a3….]

最高位称为符号位，B2U的转换规则如下：

对于B2T([1011]) : value = 1 * (-2^3) + 1 * 2^1 + 1 * 2^0

最高位为1或0保证了结果一定为负数或正数。

最值：

  • 最大值：

字长为N的无符号数：最大值即为2^N - 1;

字长为N的有符号数：最大值即为2^(N-1) - 1；

  • 最小值：

有符号的最小值：形式均为：10000..000

注意-1的补码表示与无符号的最大值相同：11111….1111;

数据类型之间的转换：

较小转变为较大：

1，对于无符号数，在前位补0即可；

2，对于有符号数，在前部补最高位的数字：

如：B2T[a1, a2, a3, ….., an]；

转变为B2T[a1, a1…. a1, a1, a2, …, an];

较大转变为较小：,

1，会造成数据的阶段，多的位数被舍弃；(类似于基于二进制的取模运算)

2，对于有符号数，采取的措施是：

先将数位看成无符号数进行截断后，在将剩余的位数转换为有符号数。

位操作运算：

按位位级操作：

支持&，|，^(异或)，~，按二进制下的一位位进行比较；

异或操作的特点：

由于

0 ^ 0 = 0; 1 ^ 0 = 1; 故数x异或全0等于其本身；

0 ^ 1 = 1; 1 ^ 1 = 0; 故数异或全1等于取反。

规范化操作—-(!!X)：

将非0数统一转化为1；

将0统一转化为0；

C语言中的逻辑运算：

逻辑运算考虑的对象不是一位位的，将原数转化为十进制，为0即返回0，非0均返回1；

&&和||与他们对应的&与|的区别：

若&&和||对第一个参数求值就能确定答案，那么就不会对第二个参数进行计算：

a && 5 / a, 不会造成0除；

p && *p++,不会造成空指针的引用；

移位运算：

左移即是将低位填充为0；

右移分为逻辑右移和算术右移；

对于无符号数进行的是逻辑右移，在高位补位的是0；

对于有符号数进行的是算术右移，在高位补位的是1；

对于移位运算，当移动的位数超过限度时：

移动位数过大时：

对于w位的数据类型，若要移动k >= w位时，

实际移动量实际上 = k mod w

移动的优先级问题：

实际上加法和减法的优先级要高于移动操作：

1 << 2+3<<4实际等价于：1<<5<<4

浮点数：

浮点数在二进制中的表示：

小数点之前每一位表示2的正幂，之后每一位表示2的负幂

例如：

观察得到：

小数点右移即是 * 2；左移即是 / 2.

局限性：
  • 这种形式无法表示出全体实数、
  • 小数点右移—–> 表示范围变大，但精度减小；
  • 小数点左移——> 表示范围变小，但精度变大；
  • 同时计算机所预留的位数是有限的；

标准化的表示：

统一形式为：

(−1)sM2E(-1)^sM2^E(−1)​s​​M2​E​​

  • s为符号位数字；
  • M为在[1.0, 2.0]之间的某个值，称为尾数，即科学计数中的小数部分
  • E表示将位数扩大到2的多少倍，称为阶码
在二进制中的编码：
  • 在32位的单精度中：
符号位(1bit) 阶码编码(8-bits) 尾数编码(23-bits)
  • 在64位的双精度中：
符号位(1bit) 阶码编码(11-bits) 尾数编码(52-bits)
阶码中正负性的表示：

对于阶码，若用补码表示——偏置编码法：

-3 ——> 101 ——-> 5 (Unsigned的读取)

3 ——–> 011 ——–> 3

对于机器，想要比较阶码部分的大小，不能直接读取，先必须判断符号位，增加了冗杂度。

故引用了偏置值(Bias)的方法来表示正负：

将阶码统一加上一个固定的数，使他们统一变成正数，在读取是直接减去固定值即可。

偏置值的规定：

对于k位的阶码字段，偏置值是：
Bias=2k−1−1Bias = 2^{k-1} - 1Bias=2​k−1​​−1
存储的阶码值EXP = 实际值E + Bias。

使得我们可以将整个浮点数当做无符号数进行比较即可。

EXP的最值：
  • 标准化：

最小值：0000….00000, 即为0；

最大值：1111…..11110, 即为2^k - 1。(不能是全为1，全为1是用来表示无限值或者NAN的)

故实际阶数的范围为：【-126 ~~ 127】(Normalize)

  • 非标准化：

值 = 0.frac * 2^(-126)

​ = frac * 2^(-126) * 2^(-23)

​ = frac * 2^(-149)

有尾数部分最大值约等于 2^23 - 1;

故实际阶数范围为：【-149, -127】。恰好实现了两者的完美衔接。

尾数的隐含编码法：

由于尾数我们将其总是调整为：1.xxxxxxxx。

故将最高位固定为1后，隐藏，以省略一位Bit。

只把小数点后部分加入尾数字段。

故最小值为 .000000…. = 1;

最大值为：.111111….1 = (2 - o) [无限接近于2]

转换实例：

对于123.0单精度的编码：

1，二进制编码为：1111011

2，转换为标准形式：1.111011 * 2^6

3，尾数部分：111011 00…000(后续0补齐26位)

4，阶码部分: Bias = 2^7 - 1 = 127;

​ 故存储值为 EXP = 6 + 127 = 133 = 10000101

5，编码为：

0 10000101 111011 00…000
漏洞：

但上述的标准化方法无法表示0，在表示完某一个最小的数值后，会直接跳到0，故引入另一种非标准化的编码方法：

非标准形式：

尾数部分：

采用0.xxxx…xx的形式。

阶码部分：

E 固定为 1 - Bias。

阶码字段全为0，

Notice：由于符号位的独立性，所以实际会出现正负0的情况。

特殊值的表示：

  • 当exp = 111…111，且frac = 000…0时，表示无穷大。

常出现在加法乘法的溢出操作中。

  • 当exp = 111…111，且frac 不等于000…000时，表示非数字，

没有答案的情况。如取-1的平方根。即NaN。

可视化的表示：

浮点数的基本操作：

类型转换的基本思想：

先得到x + y在无穷数位上的准确值，再通过舍入的操作，得到规定数位上的近似值。

例如：

int —> 32位；float —–> 32位(有效数据位：24位)；double —-> 64位；

int转换为float必然要进行数据上的舍入，而int转换为double再转换为int则可实现保留精度。

舍入原则：

  • 向0舍入:、统一舍去小数点后数据；

  • 向下舍入：向临近的更小整数取；

  • 向上舍入：向临近的更大数取；

  • 向偶数舍入：(IEEE的默认舍入方法)

中间值的定义：

对于xxxx.xxxA…..，我们需要从A开始向后取舍，那么当A…….为5000000…时，即称此时为中间值。

二进制下的中间值：即为 ……1000000…0的形式。

对于非中间值，按照自然四舍五入进行；

对于中间值，向离其最近的偶数舍入。

优势：

在大量统计计算过程中，避免了统一向上取舍造成的平均值增大的现象。

两数相乘：

假设有
(−1)s1M12E1∗(−1)s2M22E2(-1)^{s1}M_{1} 2^{E1} \quad *\quad (-1)^{s2}M_{2}2^{E2}(−1)​s1​​M​1​​2​E1​​∗(−1)​s2​​M​2​​2​E2​​
符号位s: s1 异或 s2；

尾数位M：M1 * M2；

阶码为E：E1 + E2；

两数相加：

(−1)s1M12E1+(−1)s2M22E2(-1)^{s1}M_{1} 2^{E1} \quad +\quad (-1)^{s2}M_{2}2^{E2}(−1)​s1​​M​1​​2​E1​​+(−1)​s2​​M​2​​2​E2​​

符号位与尾部均遵守正常加法运算；

阶码为取MAX(E1, E2).

浮点数不支持交换律：

存在的误差正是由于取舍操作产生的。

例如a = 1e10; b = 1e-10; c = 0.005;

若执行：(a +b) - c, 则得到0.005；

若执行a + (b -c ),由于b与c相差太大，在进行舍入时可能会忽略掉c的值，从而得到结果0；

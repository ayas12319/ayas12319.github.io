树的重心：

定义：

对于树中节点u，删除u后树分裂中若干子树，设这些子树中节点最大值为max_comp(u)， 则使得max_comp(u)最小的节点即是树的重心。

性质：

  • 删除重心u后得到的所有子树大小均 <= N / 2;

  • 树上的所有点到重心的最短距离和最小。

算法流程：

任取一点u，若以u为重心，则可将生成的子树分为两类：

一类是u的子树，一类是u上面的部分；

算出u的最大子树大小和上面部分的大小后，比较得到最大值；

参数设置：

时间复杂度为O(N)，以线性时间找到重心位置。

主要得益于vis数组的实现，每一个节点都会遍历一次，故是O(N)的复杂度。有对于每一个节点的每一条原始边，又均会被遍历一次，故为O(N) + O (N - 1) = O(N)的复杂度。

LCA(最近公共祖先)

定义：

即是两个节点的公共祖先中，距离根最远的那个。

性质：

  • lca({u}) = u;
  • 若b不为a的祖先，a不为b的祖先，则a，b必定在lca(a, b)的两颗子树上。
  • 两点集并的lca，就是两个点集各自的lca的lca；
  • 设d为树上两点间的距离，h为某点到根的距离。则有：

​ 𝑑(𝑢,𝑣) =ℎ(𝑢) +ℎ(𝑣) −2ℎ(LCA(𝑢,𝑣))

证明：

d(u, v) = d(u, lca) + d(lca, v)。

d(u, lca) = h(u) - h(lca);

所以原式等于：d(u, v) = h(u) + h(v) - 2h(lca);

求法：

  • 朴素求法：

即是dfs遍历全树，选择从深度较大的节点开始。不断对其向上调整，最终相遇位置即为lca位置。

时间复杂度：

由于是遍历任意树的路径，故需考虑退化至链表的极端情况。故时间复杂度为O(N)。

树链剖分：

概念定义：

  • 重子节点：

表示一个节点的所有子节点中子树最大的子节点，就称为重子节点。当所有子节点大小均相同时，任意选取。

定义轻子节点为剩下的所有子节点；

  • 重边：父节点和重子节点连成的边；

  • 重链：

    • 由多条重边连接而成的路径；
    • 整棵树会被剖分为若干条重链；
    • 轻儿子一定是每条重链的顶点；
    • 任意一条路径被切分为不超过logN条重链。
    • 每一个独立的轻儿子独立成为一条重链。

证明：

1，从u到v被分为从lca(u, v)分别向下走。

2，从根节点到叶节点，每经过一条轻边，树的大小就会至少减少1 / 2，假设有N个节点，经过k条轻边，则有N / 2^k >= 1，所以解有，k <= logN。

3，又有重链数量 = 轻边数量 + 1：

每一条重链的顶点必然存在一条轻边连接顶端与其父节点。但是根节点没有父节点与之相连。

4，所以每条路径经过的重链数量 = 经过的轻边数量 + 1 <= 2(logN + 1) = O(logN)。

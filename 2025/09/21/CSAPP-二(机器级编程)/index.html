再通过链接器将不同文件融合在一起(你的文件，编译文件，库文件)

操作数指示符：

指令中的操作数指出该操作要使用的源数据值：可以是给定的常数，或是从内存或寄存器中读取出来；要存放值的位置：内存或是寄存器。

操作数被分为三种类型：

  • 立即数，用来表示常数值：

写法：$ + 标准C表示的整数。

例如：$ + 地址 。表示的就是该地址，而不是取其中的数值。

  • 寄存器，用来表示存储在某个寄存器中的内容

    寄存器中通常存放的是地址。单纯的操作 %rax，提取的是存放在该寄存器中的地址；(%rax)提取的是存放在该地址处的数值。

  • 内存引用：根据计算出来的地址，访问某个位置的数据。

数据传送指令：

MOV类：

MOV类由四条指令组成：

指令 数据大小 适用寄存器 对高位的影响
movb 1字节 8位 仅改变目标8位
movw 2字节 16位 改变目标低16位
movl 4字节 32位 改变低32位，同时高32位清零
movq 8字节 64位 改变全部64位

主要语法：

源操作数是一个立即数，就是出现在指令中的数据本身，且存储在寄存器中或内存中。

目的操作数指定一个位置，要么是寄存器，要么是一个内存地址。

扩展数据方式：

当我们把1，2字节的数据移动到2字节，或4字节的寄存器中时，目标寄存器有多余的空间需要进行填充。

这里存在零扩展与符号扩展两种方式：

  • MOVZ类(零扩展)：

将目标寄存器的剩余字节全部填充为0；

  • MOVS类(符号扩展)：

通过复制源操作数的符号位来进行填充；

每条指令的最后两个字符分别是源数据的大小与目标数据的大小。

一元和二元操作：

第一个操作数可以是立即数，寄存器或是内存位置。

第二个操作数可以是寄存器或内存位置。

当第二个操作数为内存地址时，处理器必须从地址中读出值

移位操作：

移位操作中的移位量可以是一个立即量，也可以存放在寄存器%cl中

（注意，移位量能且只能放在寄存器%cl中）

%cl寄存器本质是%rcx寄存器的最低8位。

在x86-64中，移位操作对w位的数据值进行操作，移位量是由寄存器%cl的低m为决定的。这里有2^m = w，高位会被忽略。

条件码：

用来描述最近的算术或逻辑操作的属性。可以检测寄存器来执行if分支命令

各个寄存器的含义：

  • %rax

存放临时数据：特别是函数的返回值和算数计算的中间结果

  • %rsp

存放着一个内存地址，指向当前函数的栈的顶部

  • %rip

存放着一个内存地址，指向下一条要被执行的指令。

条件码的类别：

存放上一条指令中执行结果的特定状态：

1，CF(进位标志)：进位标志，无符号数的溢出或借位,

2，ZF(零标志)：最近的运算结果是否为0

3，SF(符号标志)：最近的运算结果的符号(针对于有符号数),

4，OF(溢出标志)：有符号数运算是否发生了溢出

设置条件码：

  • leaq指令不改变任何的条件码，因为其只是进行单纯的地址计算。

  • 还有两类命令，只改变条件码，而不改变任何其他值：

    • cmp指令：

      隐含着sub指令运算，但是不改变指令，改变的是状态标志。

      cmp b, a 即为 计算a - b

    • test指令：

      test b, a 类似于 计算a & b，同样不改变原式数值

  • 指令的后缀含义：

    • XXXq：表示为64位操作数；
    • XXXw：表示为16位操作数；
    • XXXb：表示为8位操作数；
    • XXXl：表示为16位操作数。

现代处理器的流水线原理：

处理器不是一条一条执行指令的，而是在同一时刻执行者不同的处理阶段：

取址 → 译码 → 执行 → 访存 → 写回

当遇到jmp这样的跳转指令时，会进行分支预测：

是继续执行下一条指令，还是跳转的目标地址。若是猜错了，则需要清空流水线，会造成15-30个时钟周期的浪费。

而基于数值传送的版本没有jmp指令的使用，即不会造成分支预测的状况。只是会计算两个分支的值，计算成本较高。无论测试的数据是什么，编译使用条件的代码均是大约8个时钟周期。控制流不依赖于数据。

但当执行函数过于复杂时或者测试结果之一会造成空指针引用等错误时，推荐或必须使用分支语句来实现。

预测错误时的罚时计算：

假设执行代码耗时TA，罚时TB，平均运行时间TC(预测成功概率为p = 0.5)：

TC = p(TA + TB) + (1 - p) TA = TA + pTB.

代入p = 0.5，计算可得TB = 2(TC - TA).

运行时栈：

以函数P调用函数Q为例：

Q在执行的过程中，P以及向上追溯的所有调用链都会暂时被挂起；

当Q运行结束返回时，任何它所分配的局部变量都会被释放。

因此，栈这种后进先出的数据结构正好对应这种内存管理机制。

栈帧的概念：

当程序运行的的内存超出寄存器的大小时，就会借助栈上的存储空间，将这部分存储空间成为栈帧。

对于函数P调用函数Q的例子：

函数P开始调用函数Q时，会将返回地址压入栈中。这个返回地址指明了函数Q返回时，接着从函数P的哪个位置继续执行。

返回地址的压栈操作并不是由push实行的，而是由call指令实现的。

例如

call指令的作用：

1，将调用函数的地址写入到程序指令寄存器%rip中，实现函数的调用；

2，将主函数中的下一个指令地址，即0x700压入栈中。

当Q函数执行结束后，0x700又会重新回到%rip中。

栈随机化：

栈的位置在每次函数运行时都会发生改变。

每次运行时，程序的不同部分都会被加载到不同地方。

增加系统的安全性，降低病毒的传播速度。

栈破坏检测：

每次程序运行时，会随机在缓冲区与返回地址之间生成一个特殊值。

每次检测这个特殊值是否被改变，以判断栈内容是否被破坏。

数据的存储：系统的本质是协调数据在磁盘与内存之间的移动。数据的存储结构总是层状的： (CPU Register —> CPU Caches(cpu缓存) —-> DRAM(内存) —-> SSD(机械硬盘) —-> HDD ——> Network Storage) 在不同的层次上决定了使用什么数据结构与算法。 在DRAM之前的结构都被称为易失性结构( 拔掉电源后，数据立刻消失)；之后的结构都被称为非易失性结构。 易失性与非易失性结构的区别： 易失性结构都是通过字节寻址的，可以随意转移到目标数值； 非易失性结构都是块状寻址：不能访问任意偏移地址，只能以块为调位调入整个页面。比如：一次性调入4000字节，以来检索到包括的目标数值。 PS: 这里我们统一将非易失性结构统称为磁盘。 数据的访问方式：顺序访问VS随机访问 随机访问：不断移动到不同的位置，读取目标的数值，访问任意位置的离散的数据。 顺序访问：给定目标数值的附近区域，一次性提取出连续的正片区域的数值。访问连续位置的块数据。 访问速度比较： 由于物理上的移动速度很慢，访问不同物理位置的开销更大...

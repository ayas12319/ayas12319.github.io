C语言转换为机器语言的流程：

先通过编译器(gcc, -Og, -S)转换成汇编语言，

再将汇编语言通过(gcc, as)转换为实际的字节形式

再通过链接器将不同文件融合在一起(你的文件，编译文件，库文件)

汇编代码的特性：

1，与硬件的架构紧密相连

2，基于寄存器操作：寄存器不是按地址访问的，是按一个个的名称访问

3，内存访问是按照地址进行的

4，通过指令集进行操作；

汇编代码的数据类型：

汇编代码中不同的数据类型主要体现在：

  • 数据大小(字节数)
  • 指令选择
  • 解释方式
  • 对于无符号数与有符号没有区别，均为一串地址；
  • 对于浮点数则在专门的register上进行操作；

反汇编器：

是一种可用于任何逆向工程的工具、

将机器代码(二进制指令)重新转换为汇编语言。但本身反汇编器无法访问.c文件亦或是.s文件，只是读取编译后的二进制文件中本身的字节序列进行转换。

访问信息：

操作数指示符：

指令中的操作数指出该操作要使用的源数据值：可以是给定的常数，或是从内存或寄存器中读取出来；要存放值的位置：内存或是寄存器。

操作数被分为三种类型：

1，立即数，用来表示常数值：

写法：$ + 标准C表示的整数。

例如：$ + 地址 。表示的就是该地址，而不是取其中的数值。

2，寄存器，用来表示存储在某个寄存器中的内容

3，内存引用：根据计算出来的地址，访问某个位置的数据。

寻址模式：

Imm(rb, ri, s)表示一个立即数偏移Imm，一个基址寄存器rb，一个变址寄存器ri和一个比例因子s

地址计算公式为：adress = Imm + R[rb] + R[ri] * s;

比例因子必须是1,2,4,8。

（register）即表示引用这个寄存器的地址

带有偏移量：

  • 对于寄存器：即为D(register);
  • 对于内存：即为Mem[Reg[R] + D];

lea命令：

根据寄存器中存储的数值，按照内存寻址的语法规则进行算数计算，将结果存储到目标寄存器中。

内存寻址的语法规则：地址 = 基址 +索引 * 比例因子。

注意：比例因子只能为1,2,4,8

只计算出ebx + 8的地址，将地址值(而不是实际数值)存入eax，不访问内存。

其他常用指令：

addq scr, dest dest = dest +src

注意：以q结尾的指令，表示操作对象是64位的数据

状态控制码：

各个寄存器的含义：

  • %rax

存放临时数据：特别是函数的返回值和算数计算的中间结果

  • %rsp

存放着一个内存地址，指向当前函数的栈的顶部

  • %rip

存放着一个内存地址，指向下一条要被执行的指令。

  • CF,ZF,SF,OF：

存放上一条指令中执行结果的特定状态：

1，CF(进位标志)：无符号数的溢出或借位,

2，ZF(零标志)：运算结果是否为0

3，SF(符号标志)：运算结果的符号(针对于有符号数),

4，OF(溢出标志)：有符号数运算是否发生了溢出

条件状态的设置：

cmp指令：

隐含着sub指令运算，但是不改变指令，改变的是状态标志。

cmp b, a = 计算a - b

test指令：

唯一目的：设置条件状态：

test b, a like 计算a & b，同样不改变原式数值

ZF set when a & b == 0;

SF set when a & b < 0;

条件状态码不能从寄存器中直接获得，但是可以通过其他指令间接获得。

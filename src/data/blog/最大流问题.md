流定义：
s-t流为一个函数f，满足：
- 对于每一条边，$0 \le f(e) \le c(e)$，即流量不超过容量；
- 对于每一个节点v（除了s-t节点），均有流守恒——流入量 = 流出量：$\sum\limits_{e \, in\, to\, v}f(e) = \sum\limits_{e \,out\, of\, v}f(e)$

割定义：
割就是把节点分成两部分(S, T)，并且s位于S中，t位于T中。

割的容量：
记为Capacity(S, T)，即离开S的边的权重之和。
割的净流量：
对于割(A, B)，其净流量定义为**从A流向B的流量 - 从B流向A的流量**。

# 最大流问题：
寻找有最大值的s-t流的划分

**流值定理：**
设f为一个流，(A, B)为其对应的一个割。则**任意流的流值就等于任意割的净流量。**
$$
\sum\limits_{out \, of \, A}f(e) - \sum\limits_{in \, to \, A}f(e) = v(f)
$$

证明：
1，对于A中除了s之外的所有顶点v，根据流守恒定理可知，上式均为0.故源点s所代表的值 = v(f)
2，对于两个端点都在集合A内部的边，其流量会相互抵消。只有从A流出到B的边，和从B流进A的边不会被抵消。

弱对偶定理：
设$f$为任意流，(A, B)为任意一割。则**流的大小不超过割的容量**。

最大流最小割引理：
设f为任意流，(A, B)为任意割。如果有：$v(f) = cap(A, B)$成立，则两者一定对应为最大流与最小割。

# 最大流算法

**朴素贪心算法：**
- 将每一条边e的流量$f(e)$初始化为0
- 寻找一条s-t路径，其中每一条边都满足$f(e) < c(e)$
- 沿着路径增广——尽可能多地往这条路径上增加流量：即取路径上的最小剩余容量。
- 重复以上操作，直到路径到达容量上限

算法局限性：
- 找到的是局部最优解$\neq$全局最优解。
- 可能多条局部较小路径，合并起来的流量较大。

**残差图：**
假设有一条初始边e，其容量为$c(e)$，当前流量为$f(e)$
定义两种边：
- 正向边：表示这条边还能继续送出多少流量：$c(e) - f(e)$
- 反向边：表示这条边最多还可以撤回多少流量：$f(e)$
综上有残差边定义为：
$$
c_{f}(e) = \left\{\begin{matrix}
 c(e) - f(e) & e \, is\, zheng\\
 f(e) & e\, is \, fan
\end{matrix}\right.
$$

**判断当前流是否为最大流算法**
- 检查当前流是否合法：
    - 检查每条边的流量是否超过容量
    - 检查除s,t外，是否满足流量守恒：$\sum\limits f_{in} = \sum\limits f_{out}$
- 建立残差图：
    - 建立正向剩余容量：$c(u,v) - f(u,v)$
    - 建立反向边，容量为：$f(u, v)$
- 从s开始进行一次bfs，若能够达到t，则说明存在增广路，当前流不是最大流；否则就是最大流。

**增广路径算法：**
沿着增广路P更新流量的过程
**伪代码：**
```
Augment(f, c, p){
  b = 路径p的瓶颈容量；————即这条路径上的最小残余容量
  if (e in E)   // 即如果e是原图中的边，则在这条边上增加流量
     f(e) = f(e) + b
  else         //如果是建立的反向边，则撤销之前的一部分流量
     f(e原) = f(e原) - b
}
```

**Ford-Fulkerson算法：**
**伪代码：**
```
Ford-Fulkerson(G, s, t, c){
  初始化每一条边的流量都为0
  构造残量网络G_f
  while(存在增广路P){
     沿着该路径增加流量：f = Augment(f, c, p)
     更新图G
  }
  return f
}
```


# 最大流最小割算法

**增广路定理：**
若没有增广路径，则说明此时的流f是最大流。

**最大流最小割定理：**
最大流的流值就等于最小割的值。

以上定理的证明可通过以下命题的相互等价证明：
1，若存在一个割(A, B),使得$v(f) = cap(A, B)$
2，流f是最大流
3，**在关于流f的残量网络中不存在增广路。**

**运行时间分析**
定理：
该算法最多在$v(f^{*}) \le nC$次迭代后终止
证明：
每一次的增广至少使得流量值增加1，源点s最多有n-1条出边，每条边的容量不超过C，所以C，所以$v(f^{*}) \le (n-1)C \le nC$

故该算法的时间复杂度为：O(mnC)

推论：
若C = 1，则该算法会在O(mn)的时间复杂度内运行

**若修改某个边上的容量，求改变之后的最大流**
- 若当前流量 < 容量：
    容量减小1后，最大流不发生改变，直接返回当前最大流量。
- 若当前流量 = 容量：
    容量减小1后，同时需要调整流量。此时需要重新建立残差图，判断是否存在s-t路径。若存在则增广1单位流后，仍可恢复最大流；否则最大流流量减小1.
    
# 增广路径的选择

在直觉上，每次选择当前剩余网络中瓶颈容量最大的那条路径，能够一次运送尽可能多的流量，从而减少总的增广次数。
但实际上，每次找到目标最宽路径的成本较高。故实际算法并不强求每次均选择最优，而是使用近似思路来逼近。

缩放参数$\Delta$
定义Gf($\Delta$)为残差网络Gf中，只保留容量$\ge \Delta$的边。

## 容量缩放算法：
伪代码：
```
Scaling-Max-Flow(G, s, t, c){
     初始化每一条边的流量为0
     初始化缩放参数Delta = 大于等于C的最小2次幂
     初始化构建残差网络
     while(Delta >= 1){
        G_f = Delta下的残差图
        while(仍然存在增广路径p){
         f = augment(f, c, P)
         update G_f(Delta) // 沿着P增广网络G_f
        }
        Delta = Delta / 2
     }
     return f
}
```
正确性证明：
由于每条边必定为整数，故有：当$\Delta = 1$时，$G_{f(\Delta)}= G_{f}$
当$\Delta$逐渐减小到1时，此时就等价于原残差图中就没有了增广路径。

运行时间分析：
- 引理一：外层循环次数为$1 + [log_{2}^{C}]$次
证明:
初始时$C \le \Delta < 2C$，每次迭代$\Delta$减半，直至$\Delta = 1$为止

- 引理二：令$f$为$\Delta$缩放阶段结束时的流，则最大流的值至多为$v(f) + m\Delta$，流的值至少为$v(f) - m\Delta$
证明：
$$
\begin{aligned}
v(f) &= \sum\limits_{e \, out \, of \, A}f(e) - \sum\limits_{e \, in \, to \, A}f(e)\\
& \ge \sum\limits(c(e) - \Delta) - \sum\limits\Delta\\
&= \sum\limits c(e) - \sum\limits \Delta - \sum\limits \Delta\\
&\ge cap(A, B) - m\Delta
\end{aligned}
$$

- 引理三：每个缩放阶段最多有2m次的增广
由于一共有O(logC)个阶段数，每个阶段增广O(m)次，每次增广耗时O(m)找到目标路径。
故总运行时间为：O($m^{2}logC$)

## 最短增广路径算法
核心思想：
选取边数最少的一条增广路径。

定理1：
找到的最短增广路径的长度是递增的。

定理2：
最多找到m条最短的增广路径，下一条的最短增广路径长度一定是增加的。

**结论:**
最短增广路径算法的时间复杂度为：$O(m^{2}n)$
**证明：**
寻找一条最短增广路径——通过BFS，耗时$O(m)$。
最多找到mn条路径：
- 对任意给定的长度，最多找到m条路径
- 对于一条路径，长度最大为n-1


# 最大流的应用
## 二分图匹配问题
### 将二分图匹配转化为网络流模型
- 在二分图G中创建虚拟源点s，虚拟汇点t。s和左半部点连边，t和右半部点连边。
- 得到新图G'，新图G'中所有新增的边的容量设为1，原有的边容量设为$\infty$——（不会成为瓶颈）
- 在G'中寻找最大流f，原二分图G中所有有流量的边即为最大匹配。
![](https://cdn.jsdelivr.net/gh/ayas12319/picture-bed@main/Snipaste_2026-06-01_17-02-06.png)

给定一个流值为k的最大流，考虑割$(L \cup s, R \cup t)$，根据流值定理得到，该割的净流量为k。
由于只有从左往右的边，故则存在一个基数为k的匹配。
所以找到最大流就等价于找到最大匹配数。

**完美匹配：**
左右部分的每一个点都能够和对应的唯一一点相匹配。

**定理一：**
设S为一组节点的子集，记$N(S)$为与S中节点相邻的所有节点组成的集合。
则若一个二分图存在一个完美匹配，则对于任意子集S，都有：$|N(S)| \ge |S|$

**婚姻定理：**
设G为一个二分图，同时满足：$|L| = |R|$. 则若对于所有的子集$S \le L$，均有$|N(S)| \ge |S|$,  则G具有一个完美匹配。

证明：
假设G没有完美匹配，即最大匹配的大小<n。
根据最大流-最小割定理：最小割的容量 < n。

设(A, B)是G'的最小割，定义：
- $L_{A} = L \cap A$
- $L_{B} = L \cap B$
- $R_{A} = R \cap A$
- $R_{B} = R \cap B$
因此割的顶点构成为：
$$
A = \{s\} \cup L_{A} \cup R_{A}, \quad B = \{t\} \cup L_{B} \cup R_{B}
$$

同时割(A, B)中不能包括原图任何边，否则不满足容量 < n。
即所有$L_{A}$中顶点的邻居，都必须在$R_{A}$中，即$N(L_{A})$包含于$R_{A}$
![](https://cdn.jsdelivr.net/gh/ayas12319/picture-bed@main/Snipaste_2026-06-01_17-45-55.png)
割$(A, B)$的容量：
$$
cap(A, B) = |L_{B}| + |R_{A}| = s \rightarrow L_{B}(edges) + R_{A} \rightarrow t(edges)
$$

同时已知：$cap(A, B) < n$，n = $|L_{A}| + |L_{B}|$，故：
$$
|L_{B}| + |R_{A}| < |L_{A}| + |L_{B}|
$$
所以得到：
$$
|R_{A}| < |L_{A}|
$$
但又有$N(L_{A})$包含于$R_{A}$,  所以得到：$|N(L_{A})| < |L_{A}|$
与前文所得结论相违背，故不成立。
